## 🚫 Why TLE?
- Time complexity ≈ O(2^n)
- Even after pruning, still exponential
- Using `set` adds extra overhead (log N insertion)

## Optimized Approach (Learned Later)
- Use Dynamic Programming
- Pattern follows Fibonacci

Relation:
dp[i] = dp[i-1] + dp[i-2]


## Key Learning
- Don’t generate all combinations when only count is required
- Always check for overlapping subproblems
- If problem says **"count valid sequences" → think DP first**
